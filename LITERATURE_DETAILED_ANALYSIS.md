- **Penulis:** 
  - Zahrotun Munawaroh (el.zahra294@gmail.com)

  - Halaman: 225-226
  - Status: Published

### Metodologi
- **Jenis Penelitian:** Kualitatif Deskriptif (Qualitative Descriptive Research)
- **Pengumpulan Data:** 
  1. Observasi (Observation)
  2. Wawancara (Interview)
  3. Dokumentasi (Documentation)
- **Lokasi Penelitian:** Regional Office Ministry of Religion (Kanwil Kemenag), Jawa Tengah Province
- **Periode:** Tidak spesifik (ketika SISKOHAT sudah beroperasi)

### Context dan Domain
- **Domain Utama:** Hajj Information System (Sistem Informasi Haji Terpadu)
- **Sektor:** Public Service / Government Service
- **Organisasi Fokus:** Kementerian Agama (Ministry of Religion)
- **Skala:** Regional (Jawa Tengah Province) dengan integrasi nasional
- **Masalah Dasar:**
  - Waiting list keberangkatan haji mencapai 19-20 tahun
  - Jumlah pendaftar di Jawa Tengah (2015) mencapai 458.650 jiwa
  - Sistem manual sudah tidak efektif untuk volume besar

### Temuan Utama (Findings)
1. **Efektivitas Implementasi SISKOHAT:**
   - SISKOHAT implementation SANGAT EFEKTIF dalam meningkatkan kinerja pelayanan haji
   - Berhasil mengatasi waiting list yang berkepanjangan
   - Meningkatkan efisiensi operasional secara signifikan

2. **Arsitektur Sistem SISKOHAT:**
   - Terintegrasi dengan 17 BPS BPIH (Kelompok Penyelenggara Ibadah Haji)
   - Host pusat di Kementerian Agama Pusat
   - Jaringan nasional mengintegrasikan kantor di seluruh Indonesia

3. **Fitur dan Proses SISKOHAT:**
   - Pendaftaran dan penyimpanan database jamaah dan petugas haji
   - Pemrosesan dokumen paspor dan visa
   - Penerbitan Dokumen Administrasi Perjalanan Ibadah Haji (DAPIH)
   - Pembayaran BPIH secara online oleh BPS BPIH
   - Sistem akuntansi BPIH terintegrasi
   - Penyusunan pramanifest (pramanifest kolter)
   - Monitoring penerbangan real-time
   - Monitoring kesehatan haji
   - Monitoring operasional haji di Tanah Air dan Arab Saudi

4. **Faktor Pendukung dan Penghambat:**
   - Faktor pendukung: dukungan pemerintah, infrastruktur teknologi, komitmen organisasi
   - Faktor penghambat: (disebutkan tetapi tidak detail dalam abstrak)

### Kontribusi Penelitian
- Bukti empiris bahwa sistem informasi terintegrasi efektif untuk pelayanan haji skala besar
- Model SISKOHAT dapat menjadi referensi untuk digitalisasi layanan publik
- Demonstrasi integrasi multi-stakeholder (17 BPS BPIH) dengan efektif

### Limitasi Penelitian
1. **Scope Geografis:** Hanya fokus pada Jawa Tengah Province, belum mencakup seluruh Indonesia
2. **Metodologi:** Penelitian kualitatif deskriptif - tidak ada pengujian hipotesis statistik
3. **Pengukuran:** Tidak ada metrik kuantitatif terukur tentang peningkatan kinerja
4. **Kepuasan Pengguna:** Tidak ada data tentang satisfaction level jamaah atau stakeholders
5. **Waktu Penelitian:** Snapshot pada saat tertentu, bukan long-term study
6. **Aspek Teknis:** Tidak membahas detail teknis infrastruktur atau security

### Rekomendasi Penggunaan untuk WS-03
- **Relevance:** TINGGI - Studi implementasi nyata
- **Maturity Level:** High - Sistem sudah operational dan proven effective
- **Implementasi Status:** Already deployed
- **Use as:** Primary case study untuk successful hajj IS implementation

## JURNAL 2: Zachman Framework for Hajj Service Architecture

**File:** `4_paper+140 (1).pdf`

### Identitas
- **Judul:** Aplikasi Web Informasi Layanan Haji pada Kementerian Haji dan Umrah Kota Sukabumi dengan Menggunakan Framework Zachman
- **Penulis:**
  - Zainul Bahar* (zainul.bahar_si24@nusaputra.ac.id) - corresponding author
  - Sudin Saepudin
- **Institusi:** Sistem Informasi, Universitas Nusa Putra, Indonesia

  - Jurnal: SINTA Jurnal Sistem Informasi dan Teknologi Komputasi
  - Volume: 3
  - Nomor: 1
  - Tahun: 2026 (Januari - SANGAT TERBARU!)
  - Halaman: 24-35
  - DOI: https://doi.org/10.61124/sinta.v3i1.140
  - ISSN: 3032-5072
  - Status: Online published January 2026

### Metodologi
- **Framework Utama:** Zachman Framework (Enterprise Architecture Framework)
- **Pendekatan:** Architecture Analysis and Design
- **Perspektif Zachman yang Digunakan:**
  1. Perencanaan (Planning)
  2. Kebutuhan Bisnis (Business Requirements)
  3. Desain Sistem (System Design)
  4. Model Teknis (Technical Models)
- **Output Utama:** Entity Relationship Diagram (ERD), Service Strategy, Process Maps, Data Flow
- **Tingkat Maturity:** Analysis & Design (bukan implementation)

### Context dan Domain
- **Domain Utama:** E-Government / Hajj Service Information System
- **Organisasi Fokus:** Kementerian Haji dan Umrah (Ministry of Hajj and Umrah), Kota Sukabumi
- **Skala:** City/Local level (Sukabumi City)
- **Masalah yang Diidentifikasi:**
  1. Layanan informasi pelayanan haji terkendala
  2. Transparansi proses pelayanan kurang optimal
  3. Belum tersedia Aplikasi Web resmi
  4. Calon jamaah harus datang langsung ke kantor
  5. Proses komunikasi hanya pada jam kerja
  6. Keterlambatan informasi dan potensi ketidakefisienan

### Temuan Utama (Findings)
1. **Problem Analysis:**
   - Bagi calon jamaah yang bekerja, kesulitan mengakses layanan
   - Menimbulkan keterlambatan informasi dan potensi ketidakefisienan
   - Kurangnya transparansi dalam proses pelayanan

2. **Solusi Menggunakan Zachman Framework:**
   - Framework mampu memetakan kebutuhan dari berbagai perspektif
   - Menghasilkan gambaran menyeluruh arsitektur sistem
   - Mencakup strategi pelayanan (service strategy)
   - Proses bisnis (process flows)
   - Alur data (data flows)
   - Identifikasi aktor (actors/stakeholders)
   - Desain model informasi (information models)

3. **Expected Outcomes:**
   - Peningkatan kualitas pelayanan publik
   - Percepatan akses layanan informasi haji
   - Efisiensi internal pada Kementerian Haji dan Umrah Sukabumi
   - Peningkatan kepuasan dan kepercayaan masyarakat

### Kontribusi Penelitian
- Demonstrasi penggunaan Zachman Framework untuk e-government service
- Pemetaan lengkap kebutuhan arsitektur untuk hajj service
- Blueprint untuk implementasi web application hajj services
- Pendekatan enterprise architecture untuk layanan publik

### Limitasi Penelitian
1. **Stage of Development:** Masih pada tahap analysis & design, BUKAN implementasi
2. **Scope Geografis:** Hanya Kementerian Haji dan Umrah Kota Sukabumi (satu lokasi)
3. **Empirical Data:** Tidak ada data empiris hasil implementasi atau testing
4. **Validation:** Architecture belum divalidasi dengan implementation
5. **Sustainability:** Tidak ada diskusi tentang maintenance atau evolution

### Rekomendasi Penggunaan untuk WS-03
- **Relevance:** TINGGI - Framework approach untuk hajj IS
- **Maturity Level:** Medium - Konseptual/Design phase
- **Implementasi Status:** Proposed architecture (not yet deployed)
- **Use as:** Reference untuk metodologi enterprise architecture design

## JURNAL 4: McCall Quality Analysis of SIHAT System

**File:** `Analisis-Kualitas-Sistem-Informasi-Haji-Terpadu-Menggunakan-Metode-McCall.pdf`

### Identitas
- **Judul:** Analisis Kualitas Sistem Informasi Haji Terpadu Menggunakan Metode McCall
- **Penulis:**
  - Ahmad Farisi¹ (ahmadfarisi@mdp.ac.id)
  - Rizani Teguh² (rizani_teguh@mdp.ac.id)
  - Ratih Lestari³ (ratih.lestari@mhs.mdp.ac.id)
- **Institusi:** Program Studi Sistem Informasi, Fakultas Ilmu Komputer dan Rekayasa, Universitas Multi Data Palembang
- **Publikasi:**
  - Jurnal: JOINTECS (Journal of Information Technology and Computer Science)
  - Volume: 7
  - Nomor: 2
  - Tahun: 2022
  - Halaman: 83-92
  - Diterima: 24-05-2022
  - Revisi: 28-05-2022
  - Published Online: 31-05-2022
  - ISSN: e-ISSN:2541-6448, p-ISSN:2541-3619
  - Website: publishing-widyagama.ac.id

### Metodologi
- **Framework Kualitas:** McCall Software Quality Model
- **Metode:** Quantitative survey-based assessment
- **Sampel:** 30 responden
- **Variabel Kualitas McCall:**
  1. **Correctness** - Ketepatan fungsi
  2. **Reliability** - Keandalan sistem
  3. **Efficiency** - Efisiensi kinerja
  4. **Integrity** - Integritas data
  5. **Usability** - Kemudahan penggunaan
- **Proses Pengukuran:**
  - Distribusi kuesioner McCall ke 30 responden
  - Testing validitas dan reliabilitas
  - Pembobotan variabel & indikator oleh expert
  - Skala pembobotan: 0.1 sampai 0.5
- **Analisis Data:** Weighted scoring based on expert input

### Context dan Domain
- **Domain Utama:** Software Quality Measurement / System Quality Assessment
- **Sistem yang Diukur:** SIHAT (Sistem Informasi Haji Terpadu) Arraudhah
- **Organisasi:** PT. Arraudhah Wisata Imani (Hajj Service Provider)
- **Karakteristik SIHAT:** Sistem informasi terpadu untuk layanan haji khusus
- **Fokus:** Mengukur quality level sistem yang sudah operational

### Temuan Utama (Findings)
1. **Quality Measurement Results:**
   
   | Factor | Score | Rating |
   |--------|-------|--------|
   | Correctness (Ketepatan) | 58% | Good |
   | Reliability (Keandalan) | 30% | Low-Medium ⚠️ |
   | Efficiency (Efisiensi) | 19% | Poor ❌ |
   | Integrity (Integritas) | 58% | Good |
   | Usability (Kemudahan) | 45% | Medium |
   | **OVERALL QUALITY** | **41%** | **SUFFICIENT** |

2. **Overall Quality Assessment:**
   - Overall quality value: 41%
   - Range: 41%-60%
   - Conclusion: SIHAT Arraudhah is of SUFFICIENT QUALITY (cukup berkualitas)

3. **Analysis Insights:**
   - Correctness dan Integrity yang baik menunjukkan fungsionalitas dasar OK
   - Reliability rendah (30%) = sistem tidak konsisten, sering error/down
   - Efficiency sangat rendah (19%) = performance issues, sistem lambat/tidak responsive

### Kontribusi Penelitian
- Evaluasi empiris kualitas sistem haji terpadu dengan metodologi standar
- Identifikasi area weakness dalam SIHAT implementation
- Bukti bahwa sistem operational tetapi masih banyak improvement needed
- Referensi untuk quality assessment hajj IS di provider lain

### Limitasi Penelitian
1. **Sample Size:** Hanya 30 responden (relatively small)
2. **Scope:** Satu provider saja (PT. Arraudhah Wisata Imani)
3. **Tidak Representatif:** Tidak mencakup KBIH reguler atau provider lain
4. **Missing Details:**
   - Tidak ada rekomendasi spesifik untuk improvement
   - Tidak dijelaskan root cause dari low scores
   - Tidak ada action plan
5. **Framework Age:** McCall method adalah framework lama (developed 1970s)
   - Tidak mencakup aspek modern: security, scalability, maintainability
6. **Snapshot Analysis:** Hanya measurement saat penelitian, bukan trend analysis

### Rekomendasi Penggunaan untuk WS-03
- **Relevance:** TINGGI - Quality assessment of operational hajj IS
- **Maturity Level:** Medium - Operational system with quality issues
- **Implementasi Status:** Deployed but with performance/reliability concerns
- **Use as:** Benchmark untuk quality measurement, identifies improvement areas

**File:** `hh,+FINISH+-+Edhy.pdf`

### Identitas

- **Penulis:**
  - Zainal Mustaqim¹ (zainalmustaqim267@gmail.com)
  - Edhi Prayitno²* (edhi.epo@nusamandiri.ac.id) - corresponding author
- **Institusi:** Program Studi Sistem Informasi, Universitas Nusa Mandiri
  - Lokasi: Jl. Jatiwaringin No. 2, Cipinang Melayu, Jakarta Timur
- **Publikasi:**
  - Jurnal: Jurnal Innovation and Future Technology (IFTECH)

  - Tahun: 2024 (SANGAT BARU!)
  - Halaman: 61-62
  - P-ISSN: 2656-1719
  - E-ISSN: 2656-2774
  - Status: Published

### Metodologi
- **Software Development Method:** Waterfall Method
- **Metode Pengumpulan Data:** 
  - Direct observation (observasi langsung)
  - Interviews dengan admin dan ketua KBIH
- **Framework & Tools:**
  - Backend Framework: CodeIgniter (PHP Framework)
  - Database: MySQL
  - Platform: Web-based
- **Tahap Waterfall:** Requirements → Design (research fokus pada design phase)
- **Tingkat Maturity:** Design proposal (bukan implementasi)

### Context dan Domain
- **Domain Utama:** Hajj and Umrah Registration System / KBIH Service
- **Organisasi Fokus:** KBIH Ibnu Aqil
- **Parent Organization:** Yayasan Haji Agus Salim Bogor
- **Lokasi:** Bogor, Indonesia
- **Karakteristik KBIH Ibnu Aqil:**
  - Bergerak di bidang pembinaan dan bimbingan jamaah haji dan umrah
  - Pengakuan: Keputusan Menteri Agama No. 317 Tahun 2002
  - Layanan: Registrasi, bimbingan manasik haji, guidance ke tanah suci

### Temuan Utama (Findings)
1. **Current Problems Identified:**
   - Sistem pendaftaran KBIH Ibnu Aqil masih MANUAL
   - Jamaah harus datang langsung ke lokasi untuk registrasi
   - Informasi detail proses pendaftaran tidak mudah diakses
   - Informasi biaya (financial details) tidak transparan
   - Jadwal keberangkatan haji/umrah tidak mudah diketahui
   - Proses pembuatan laporan pendaftaran sangat lama
   - Pelaporan masih dilakukan manual dengan Microsoft Excel
   - Data entry bertahap = membuang banyak waktu

2. **Proposed Solution:**
   - Web-based registration information system
   - Online accessibility untuk calon jamaah
   - Penyediaan informasi lengkap tentang:
     - Detail proses pendaftaran
     - Biaya (costs incurred)
     - Jadwal keberangkatan
   - Automated report generation (bukan manual Excel)
   - Integration jamaah data dengan sistem

3. **Expected Outcomes:**
   - Meningkatkan layanan kepada jamaah (improved services)
   - Solusi untuk masalah yang ada
   - Efisiensi operasional KBIH
   - Aksesibilitas informasi lebih baik

### Kontribusi Penelitian
- Desain sistem registrasi haji/umrah untuk KBIH Ibnu Aqil
- Aplikasi metodologi Waterfall untuk hajj IS design
- Menggunakan CodeIgniter + MySQL (teknologi praktis)
- Solusi untuk KBIH reguler (bukan khusus/special)
- Focus pada user convenience dan operational efficiency

### Limitasi Penelitian
1. **Development Stage:** Masih DESIGN phase, bukan implementasi
   - Tidak ada testing hasil design
   - Tidak ada validation dengan users
   - Tidak ada proof of concept

2. **Scope:** Terbatas pada KBIH Ibnu Aqil (satu organisasi)
   - Tidak generalizable ke KBIH lain
   - Spesifik untuk Bogor

3. **Technical Limitations:**
   - Waterfall method adalah sequential - mungkin kurang fleksibel untuk hajj domain yang dinamis
   - Tidak ada diskusi tentang security considerations
   - Tidak ada privacy/data protection discussion
   - Scalability tidak dibahas

4. **Research Gaps:**
   - Tidak ada user requirement specification detail
   - Tidak ada ERD atau detailed database design
   - Tidak ada security analysis
   - Tidak ada discussion tentang integration dengan government systems

5. **Publication:** Sangat baru (2024) tapi masih proposal stage

### Rekomendasi Penggunaan untuk WS-03
- **Relevance:** MEDIUM-HIGH - Recent design for KBIH registration
- **Maturity Level:** Low - Design proposal (no implementation evidence)
- **Implementasi Status:** Not yet deployed (design only)
